# AI-project 

## 1. 프로젝트 개요

### 주제 
실제 블랙박스 환경을 위한 YOLOv11 기반 도로 위험물 탐지 고도화 및 데이터 환경 차이에 따른 전처리 한계 분석

### 문제 정의 및 개발 동기
* **실전 도로 위의 치명적인 위험 요소:** 도로 표면의 포트홀(Pothole), 균열(Crack) 및 관리되지 않은 맨홀(Manhole) 영역은 차량 파손을 넘어 운전자의 시야가 제한되는 주행 환경에서 심각한 대형 2차 교통사고를 유발하는 주된 위험 요인입니다.
* **기존 AI 모델 및 공개 데이터셋의 한계:** 현재 객체 탐지 알고리즘 연구에 널리 활용되는 공개 데이터셋은 대부분 차량이 정지된 상태이거나 깨끗하게 촬영된 'A급 이미지' 위주로 구성되어 있습니다. 이로 인해 실제 블랙박스나 주행 환경처럼 진동, 속도, 조도 변화가 심한 실생활 시스템에 적용할 때는 탐지 성능이 현저히 떨어지는 한계가 존재합니다.
* **실전 주행 데이터 검증의 필요성:** 본 프로젝트는 이러한 공백을 메우기 위해, 단순 정지 상태가 아닌 **실제 주행하는 차량의 블랙박스 화면과 유사한 동적 데이터 환경**에서도 도로 파손 부위를 안정적으로 분류하고 실시간으로 추적할 수 있는지 실증적으로 검증하고자 시작되었습니다.

### 연구 목적 및 개발 목적
* **데이터 도메인 차이에 따른 전처리 한계 실증:** 실제 차량 주행 중 발생하는 동적 블랙박스 환경과 기존 객체 탐지 연구용 공개 데이터셋 간의 도메인 격차를 규명하고, 일반적인 전처리 기법들이 탐지 성능에 미치는 한계를 실험적으로 입증합니다.
* **전처리 및 증강 기법의 성능 영향도 비교 분석:** 객체 탐지 성능 지표(`mAP50`) 개선을 목적으로 도입한 다양한 데이터 전처리 기술(`ROI Crop`, `CLAHE`)과 데이터 증강(`Augmentation`) 기법이 실제 도로 위험물 탐지에서 왜 성능 저하를 유발했는지 정량적·정성적으로 체계적인 비교 분석을 수행합니다.
* **실전형 데이터 정제 및 검증 방향성 제안:** 정성적 비디오 매핑 검증과 한계 원인 분석(맥락 정보 손실, 노이즈 증폭 등)을 통해, 단순히 이미지 품질을 높이는 전처리가 아닌 **실제 블랙박스 주행 환경의 특성을 반영한 최적의 데이터 정제 가이드라인**을 도출하고 향후 연구 방향을 제안합니다.

데이터셋 출처 및 기본 정보
* **공식 명칭:** Road Damage Dataset: Potholes, Cracks and Manholes
* **제공 기관:** Kaggle (글로벌 오픈소스 데이터 플랫폼)

데이터 가공 및 정밀 분할 구조
* **데이터셋 가공:** 원본 도로 전경 데이터에서 탐지 모델의 효율성을 극대화하기 위해 파손 의심 영역을 정밀하게 크롭(Crop) 및 라벨링하여 데이터 완성도를 고도화했습니다.
* **정답 유출 방지 (Data Leakage Filter):** 학습을 진행하기 전, 검증 데이터셋(Validation Dataset)에 포함된 시험 문제가 학습 데이터셋(Train Dataset)에 중복 포함되어 모델이 답안을 외우는 현상을 방지하기 위해, 중복 데이터를 전수 추적하여 필터링 및 제거하는 고도화 공정을 거쳤습니다.
* **데이터셋 분할 비율:** 원본 데이터는 **Train(80%) : Validation(20%)**의 독립적인 평가 구조로 분할되었습니다.
  * **Train Dataset (학습 데이터):** 1,607장 (이미지 및 라벨 텍스트 매칭 완료)
  * **Validation Dataset (검증 데이터):** 402장

## 4. 모델 설명 및 개발 내용
실제 블랙박스 주행 데이터 환경에 대응하는 최적의 파이프라인을 도출하기 위해, 본 프로젝트에서는 `YOLOv11` 구조를 기반으로 모델 체급 확장 및 다양한 영상 처리 가공 기법을 조합한 총 5가지 실험 트랙을 설계하여 한계 분석을 수행하였습니다.

실험별 전처리 및 학습 파이프라인

* 1차 실험 Baseline 학습 (YOLOv11s)
기준 탐지 성능 지표를 수립하기 위해, 추가적인 화질 가공 없이 정제된 기본 데이터셋만을 활용하여 Small 모델로 가볍게 학습을 진행했습니다.

```python
from ultralytics import YOLO

# Baseline: YOLOv11s 가중치 기반 기본 학습 파이프라인 구축
model_base = YOLO("yolo11s.pt")
model_base.train(
    data="data.yaml", 
    epochs=50, 
    batch=16, 
    imgsz=640, 
    name="yolo11s_baseline"
)
```

* 2차 실험 Model Upgrade (YOLOv11m)
모델의 백본(Backbone) 네트워크 파라미터 체급을 확장하여, 도로 위험물의 미세 특징 추출 성능을 고도화하고자 Medium 모델로 스케일업 실험을 진행했습니다.

# Model Upgrade: YOLOv11m 모델 스케일업 학습
```python
model_mid = YOLO("yolo11m.pt")
model_mid.train(
    data="data.yaml", 
    epochs=50, 
    batch=16, 
    imgsz=640, 
    name="yolo11m_upgrade"
)
```
* 3차 실험: Data Augmentation 적
차량 주행 중 발생할 수 있는 노면의 물리적 각도 변화 및 기상 조건 변화를 강제로 모사하기 위해, 광학적·기하학적 증강 옵션을 부여하여 모델의 일반화 성능을 유도했습니다.

# Augmentation: 조도 변화 및 흔들림 가혹 환경 모사 학습
```python
model_aug = YOLO("yolo11s.pt")
model_aug.train(
    data="data.yaml", 
    epochs=50, 
    batch=16, 
    imgsz=640, 
    hsv_v=0.6,       # 명도 변형 확장
    hsv_s=0.5,       # 채도 변형 확장
    degrees=10.0,    # 미세 회전 강제
    translate=0.2,   # 평행 이동 변형
    name="yolo11s_augmentation"
)
```
* 4차 실험: ROI Crop 전처리 처리
도로 전경 이미지에서 하늘, 가로수, 건물 등 탐지와 무관한 불필요한 상단 배경을 파이썬 스크립트로 필터링하고, 실제 결함이 존재하는 노면(Region of Interest) 영역만 정밀하게 잘라내어(Crop) 데이터셋을 정제했습니다.

# ROI Crop 전처리가 반영된 데이터셋으로 학습 연동
```python
model_crop = YOLO("yolo11s.pt")
model_crop.train(
    data="crop_data.yaml",  # 크롭 전용 데이터 명세
    epochs=50, 
    batch=16, 
    imgsz=640, 
    name="yolo11s_roi_crop"
)
```

5차 실험: CLAHE 국소 명암 대비 강조 적용
아스팔트 노면과 어두운 균열(Crack) 경계선의 식별력을 극대화하기 위해, OpenCV를 활용한 국소 적응형 히스토그램 균등화(CLAHE) 알고리즘을 이미지 전반에 적용하여 대비를 극대화했습니다.

```python
import cv2
# OpenCV 활용 CLAHE 이미지 가공 파이프라인 구현 샘플
clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8, 8))
# (노트북 소스코드를 통해 전체 이미지를 변환 처리한 후 YOLOv11 학습 수행)
```

### 핵심 분석 결과 및 인사이트 (Core Insights)

* **일반적인 전처리 기법의 도메인 한계 규명:** 이미지 화질 개선 및 노이즈 제거를 목적으로 도입한 `ROI Crop`과 `CLAHE` 기법이 오히려 Baseline보다 성능을 저하시키는 역효과(Trade-off)를 확인하였습니다.
* **도로 탐지 내 '맥락 정보'의 중요성:** 도로 파손 및 위험물 탐지 환경에서는 결함 객체 고유의 형태적 특징뿐만 아니라, **도로의 원근감, 거리에 따른 소실점, 주변 노면과의 연속성 등 '맥락 정보(Context)'가 학습에 매우 중요한 요소**임을 실증적으로 증명했습니다. ROI Crop은 이 맥락을 단절시켜 오탐지를 유발했습니다.
* **노이즈 증폭 현상 분석:** 특히 국소 명암 대비를 강조하는 `CLAHE` 기법의 경우, 아스팔트 노면 고유의 불규칙하고 거친 질감마저 미세 균열(Crack)이나 포트홀의 경계선 노이즈로 오인하게 만들어 모델의 혼선과 오탐지율을 급격히 증가시키는 명확한 한계를 발견하였습니다.

### 6. 한계점 및 향후 계획 (Limitations & Future Work)

### 연구의 핵심 한계점 및 검증 전략

* **클래스 인덱스 불일치 및 정성적 우회 검증:** 본 프로젝트 진행 중, 학습에 활용된 데이터셋과 실전 테스트 데이터 간의 `data.yaml` 설정 내 **클래스 순서(Class Index Mapping)가 상호 불일치하는 현상**이 식별되었습니다. 이를 보정하기 위해 인코딩 변환 작업을 시도하였으나, 데이터셋 고유의 포맷 차이로 인해 인덱스 정렬을 완벽하게 해결하는 데 한계가 존재했습니다.
  * **우회 검증을 통한 실효성 입증:** 정량적인 Test mAP 수치 도출이 불가능해진 한계를 극복하기 위해, 본 팀은 **실환경 주행 영상(`맑은날.mp4`) 환경에서 바운딩 박스가 실제 도로 노면에 정확하게 매핑되고 추적(Tracking)되는지 직접 육안으로 확인하는 '정성적 검증'**을 도입하여 모델의 실전 효용성을 증명하였습니다.

* **기상 및 주행 도메인의 편향성 (Environmental Constraints):** 현재 모델의 추론 및 검증 파이프라인은 '맑은 날'의 비디오 스트림만을 기준으로 수행되었습니다. 흐린 날, 우천 시, 혹은 야간 주행 상황과 같이 조도가 급격히 저하되거나 노면 반사가 심한 가혹 주행 환경에서는 탐지 신뢰성을 완벽히 보장하기 어려우므로, 향후 도메인 확장을 위한 추가 데이터 확보가 필수적입니다.


### 향후 연구 계획 (Future Work)

* **클래스 인덱스 표준화 모듈 개발:** 외부 데이터셋이나 이기종 도로 환경 데이터를 결합할 때 발생하는 클래스 넘버링 충돌 오류를 원천 차단할 수 있도록, 데이터 로드 시 인덱스를 자동으로 매핑하고 동기화하는 '클래스 표준화 전처리 파이프라인'을 정형화할 예정입니다.

* **실전형 맥락 보존 경량화 및 엣지 가속화:** 단순히 ROI 영역을 잘라내어(Crop) 도로의 원근감과 소실점 등 중요한 맥락 정보를 훼손하는 방식 대신, 가중치 마스킹 기법을 활용해 주변 환경을 유지하는 전처리를 연구할 계획입니다. 더불어, 차량용 저전력 임베디드 장비나 블랙박스 환경에서도 실시간(Real-time) 연산이 가능하도록, 모델 가중치를 **NVIDIA TensorRT** 가속 포맷으로 변환하고 **양자화(Quantization, INT8/FP16)** 공정을 적용하는 경량화 연구를 이어갈 것입니다.
